# חיפוש עברי — תוכנת חיפוש שולחנית בקבצים

Hebrew Search — an offline Windows desktop app for full-text search in Hebrew
documents (PDF, Word, text, scanned images with Hebrew OCR).

תוכנת חיפוש שולחנית ל-Windows, עם דגש חזק על עברית, לחיפוש בתוך קבצים:
Word‏ (docx), ‏PDF, טקסט, ותמונות/סריקות (עם OCR עברי). כל תוצאה מוצגת
בהקשר שלה על המקור, ניתן לפתוח את הקובץ במיקום המדויק, ולייצא לטקסט.
הכל רץ מקומית (offline).

## יכולות עיקריות

- חיפוש טקסט מלא מהיר (מנוע Tantivy, דירוג BM25).
- עברית איכותית: הסרת ניקוד, טיפול באותיות סופיות, תחיליות (ו/ה/ב/כ/ל/מ/ש)
  וסטמינג קל של סיומות — כך ש"ילד" מוצא גם "הילדים".
- חילוץ מ-Word‏, PDF‏, טקסט, HTML, ותמונות.
- OCR עברי מובנה (Tesseract, מנוע ומודל עברית מצורפים ל-EXE) עם הגדרות
  מלאות מהממשק: שפות, ניתוח פריסת עמוד (PSM), רזולוציה ועיבוד-מקדים.
- ריבוי מנועי OCR: מנוע Surya 2 אופציונלי (דיוק גבוה בעברית + זיהוי
  עמודות ופריסת עמוד — מתאים לעלונים), מותקן בלחיצה מתוך ההגדרות.
- אשף אימון מודל זיהוי לפי גופן: בוחרים גופן עברי מותקן, והתוכנה מייצרת
  דפי תרגול, מאמנת מודל Tesseract מותאם ומודדת את השיפור — הכל אוטומטי.
- שימוש חוזר ב-OCR לפי תוכן: קובץ שהועבר/שוכפל מקבל את הטקסט הקיים מיד.
- תיקון אוטומטי של PDF עם טקסט עברי הפוך (סדר חזותי מקבצים ישנים) - גם
  באינדוקס וגם באינדקסים קיימים; לקבצים בעייתיים אחרים: "סרוק OCR מלא"
  לקובץ בודד או מצב "הרץ OCR תמיד" שמתעלם משכבת הטקסט.
- חילוץ טקסט מהצפיין: מהטקסט השמור / שכבת הטקסט / OCR מחדש, בטווח
  עמודים נבחר, ל-TXT או Word - עם התקדמות וביטול; ו"חלץ טקסט מאזור"
  בגרירת מלבן גם על עמודי PDF.
- זום ב-Ctrl+גלגלת וגרירת המסמך בעכבר בצפיין ה-PDF ובמציג התמונות.
- שמירת התקדמות OCR חלקית - סגירה באמצע קובץ גדול לא מאבדת את מה שזוהה.
- ייצוא וייבוא בין מחשבים (כולל מחשב ללא אינטרנט): הגדרות, אינדקס עם
  טקסטי ה-OCR, מודלים מאומנים ומנוע Surya — בקובץ אחד, בבחירת רכיבים.
- הצגת ההקשר על המקור עם הדגשה וניווט בין ההופעות.
- פתיחת הקובץ במיקום (וב-PDF קפיצה לעמוד), הצגה בתיקייה, וייצוא לטקסט.
- מעקב חי אחר שינויי קבצים (עדכון אינדקס מצטבר).
- סינון לפי סוג קובץ ותיקייה, וחיפוש ביטוי מדויק.
- יומן אבחון מובנה + קובצי לוג לאיתור תקלות.

## הרצה מהקוד (למפתחים)

דרישות: Python 3.11+‏, Node.js 18+.

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt

# בניית הממשק
cd frontend
npm install
npm run build
cd ..

# הרצה
python main.py
```

לפיתוח הממשק בזמן אמת: הריצו `python -m uvicorn backend.server:app --port 8756`
ובמקביל `npm run dev` בתיקיית `frontend` (הפרוקסי מפנה ל-8756).

## OCR

ה-EXE הארוז כולל את מנוע Tesseract, את מודל השפה העברית ואת כלי האימון —
אין צורך בהתקנה. בהרצה מהקוד: התקינו Tesseract (מומלץ בניית UB Mannheim,
כולל רכיב Training Tools עבור אשף האימון) והציבו את `heb.traineddata`
בתיקיית `tessdata` שבשורש הפרויקט (או הגדירו `TESSDATA_DIR`).

### מנוע Surya 2 (אופציונלי)

מנוע זיהוי מדויק במיוחד לעברית (90.9% בבנצ'מרק הרשמי) שמנתח גם את פריסת
העמוד (עמודות, כותרות, סדר קריאה). מותקן מתוך הגדרות ➜ זיהוי טקסט (OCR)
בהורדה של כ-3GB לסביבה מבודדת ב-`%LOCALAPPDATA%\HebrewSearch\engines\surya`.
המשקולות יורדות ממראה ב-GitHub Releases (נגיש גם מאחורי מסנני אינטרנט
כמו נטפרי). ללא כרטיס NVIDIA הזיהוי איטי (כ-2 דקות לעמוד) — מומלץ לעלונים
וחומר מורכב, לא לספרים ארוכים. במחשבים ללא NVIDIA זמינה גם האצה ניסיונית
על GPU משולב (Vulkan) — מופעלת בהגדרות, עם נפילה אוטומטית ל-CPU אם אינה
יציבה במחשב. למחשב ללא אינטרנט: השתמשו בסעיף "ייצוא וייבוא" בהגדרות.

### אימון מודל לפי גופן

בהגדרות ➜ זיהוי טקסט ➜ "אימון מודל זיהוי מותאם לפי גופן": בוחרים גופן
עברי מותקן שדומה לחומר הסרוק, והתוכנה מייצרת שורות תרגול מלאכותיות עם
הרעשה, מריצה fine-tuning על מודל העברית של Tesseract, ומדווחת את השיפור
מול מודל הבסיס. המודלים נשמרים ב-`%LOCALAPPDATA%\HebrewSearch\tessdata_custom`
וזמינים מיד לבחירה תחת "מודל שפה".

## בניית EXE

```bash
pip install pyinstaller
pyinstaller hebrew_search.spec --noconfirm --clean
```

התוצר: `dist\HebrewSearch.exe` — קובץ יחיד ועצמאי.

## היכן נשמרים הנתונים

`%LOCALAPPDATA%\HebrewSearch` — אינדקס, קטלוג (SQLite) ולוגים. ניתן לעקוף
דרך משתנה הסביבה `HEBREW_SEARCH_DATA`.

## ארכיטקטורה

- `backend/` — FastAPI, מנוע Tantivy, חילוץ, נירמול עברי, מעקב קבצים.
- `backend/hebrew_bidi.py` — זיהוי ותיקון טקסט עברי בסדר חזותי (הפוך)
  משכבות טקסט פגומות ב-PDF ומפלט OCR.
- `backend/export_service.py` — ייצוא טקסט מקובץ כמשימת רקע עם התקדמות.
- `backend/extractors/ocr_engines/` — תשתית ריבוי מנועי OCR: כל מנוע מממש
  ממשק אחיד ומתאר את הגדרותיו, וה-UI נבנה דינמית (Tesseract מובנה;
  Surya כ-worker בתת-תהליך בסביבה מבודדת).
- `backend/training/` — אשף אימון מודלים לפי גופן (רינדור, אימון, אימות).
- `backend/transfer.py` — ייצוא/ייבוא נתונים בין מחשבים בבחירת רכיבים.
- `frontend/` — React‏ (Vite), ממשק RTL, מוגש כקבצים סטטיים מתוך ה-backend.
- `main.py` — משגר: מריץ את השרת בתהליך רקע ופותח חלון pywebview.
- `_devtest/` — בדיקות עשן (רצות גם ב-CI).
